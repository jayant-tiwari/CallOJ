Creating a new problem:


  • The problem statement should be easy to understand and the problem name or problem code must not be used in any other
    problem previously made. If so, you will be prompted to use a new problem name or problem code.

  • The time limit specifies the maximum amount of time(in seconds) the solution should run before terminating.

  • The memory limit specifies the maximum amount of memory(in kB) the solution can allocate before terminating.

  • If you choose visibility as Public, the problem will be visible to everyone, but if you choose the visibility as Private
    only you can view it.

  • The problem type can be IOI or ICPC which will choose the evaluation method.
      The IOI evaluation method employs a partial marking scheme which can be configured inside the init.yml file.
      The ICPC evaluation method employs a 0/1 marking scheme which can be configured inside the init.yml file.

  • You need to copy the input and the expected output to the problem in a textfile and upload them.



    • Creating a new contest:


  • The contest name or contest code must not be used in any other contest. If so, you will be prompted to use a new name or
    code for the same.

  • After selecting the details regarding the contest duration and starting time, you will be prompted to choose a single
    marking scheme for all the problems included in the contest.
