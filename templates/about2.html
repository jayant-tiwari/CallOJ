Create a problem



Problem Code:

  • The problem code acts as an identifier to the problem. Thus, enter tag the problem by keeping the problem code
    unique and just a single word without any spaces.


    • Problem Name:

  • Just put in a fancy name there to lure the users to solve this problem


    • Problem statement:

  • The problem statement comes with a WYSIWYG editor. Please format the problem and make it explainable
    enough for users to solve the problem. Please follow the guidelines in order to make the problem more understandable.

    Note: These are just the guidelines, you need not strictly follow this if you have some creative method


  • Problem statement: Put in a statement explaining your problem
  • Input: This section conveys the user what the input will be provided by the judge
  • Output: This section conveys the output the judge expects from the user's code to produce.
    •

    Time limit:

  • Time Limit specifies how much time should each test file take to run on the judge i.e. the amount of time the
    code of the user should take to produce solutions.

  • This specifies the user how efficient the solution code needs to be and the judge in cutting of the feed and
    generating results.
    •

    Memory limit:

  • Each problem solution is given some desired space it must occupy while running the code. The memory stack that can
    be used by the solution in kilobytes (kb).


    • Visibility:

  • Each idea is a million dollar scheme! Similarily, each question's idea is also a big deal of teaching someone who does
    not know the specific concept the question conveys.

  • CallOJ gives each user to create a private question and test it. Once the question with private tag is create, it shall be
    visible in the problem section only to the user who created the problem.

  • This gives the capability to the user to create a problem and use it in the future contests.

  • NOTE: ONCE ANY PROBLEM IS USED IN A CONTEST, IT BECOMES PUBLIC AT 10:30PM IST ONCE THE CONTEST IS OVER!


    • Problem type:

  • IOI Marking: User is ranked according to the maximum score. Users with same score achieve same rank and the next ranks
    are skipped. The problem is partially graded in order to maximize the score and comes with no penalty on wrong submissions.

  • ICPC Marking: User is ranked according to the most problems solved. For awards, or in determining qualifier(s) e.g. ICPC World
    Finals, users who solve the same number of problems are ranked by least total time.
    The total time is the sum of the time consumed for each problem solved. The time consumed for a solved problem is the time elapsed
    from the beginning of the contest to the submission of the first accepted run plus 20 penalty minutes for every previously rejected
    run for that problem.
    There is no time consumed for a problem that is not solved.

  • init.yml: init.yml is required to grade the problem and specify the points each testcase beholds.
    Details about init.yml can be found here


    • Testcases:

  • The file to be uploaded should only be of .txt format. For each testcase, add test cases and input file takes
    in the input test cases and output file takes in the output test cases in a text file.
